# Session: 2026-05-19 — Initial Design Conversation

## Summary

First working session on the Hydra Swarm Framework 2.0 rewrite. This session did not produce any implementation code — it produced the knowledge base scaffold, the three philosophical pillars, the three execution modes, the V1/V2 scope boundaries, and the component breakdown that will guide all future implementation work.

## Key Outcomes

### Three Pillars Codified
1. **Intent is permanent. Code is exhaust.** (LLM Knowledge Base)
2. **No decision without verification.** (Verified Knowledge — every agent has brave-search)
3. **Code survives the machine.** (Omnidirectional Implementation — 5-state machine)

### Three Execution Modes
- `quick` — 1 agent, 1 shot, just do it
- `rigorous` — 1 agent, full 5-state machine
- `swarm` — full pipeline: Architect → N agents → Tribunal → Integrator → Librarian

### V1 Scope
- Python-only target repos
- Flat adversarial topology (single-stage tasks)
- Sandbox: `uv venv` + `pip install -e ".[dev,test]"`
- Test runner: pytest. Linter: ruff, mypy.
- Agent runtime: opencode CLI

### V2 Scope (future)
- Staged DAGs with artifact-based IPC (the wiki pattern IS the protocol)
- Multi-language via MCP/Skills (`SKILL.md` in target repos)

### Wiki Scaffold Created
- `AGENTS.md` — schema that tells any LLM how to work on this project
- `wiki/log.md` — chronological session log
- `wiki/index.md` — catalog of all pages
- `wiki/philosophy.md` — three pillars
- `wiki/architecture.md` — high-level design and component topology
- `wiki/components/*.md` — 6 component pages (one per layer)
- `wiki/versions/v1-scope.md` — V1 boundaries and success criteria
- `wiki/versions/v2-future.md` — V2 design preview
